Project overview

About the SMS security classification project

The project addresses SMS fraud and message triage by classifying incoming texts into ads, gov, notifs, otp, and spam. The web application is built with Flask only and integrates both an ensemble classifier and a neural network.

Problem

Users often receive mixed Tagalog-English SMS messages where scam, OTP, government, and promotional content can look similar. A classifier helps flag risky messages and support better decision making.

Dataset

The dataset includes sender, message body, and class label. The upgraded preprocessing pipeline now preserves sender identity as a normalized token so the model can learn source-level patterns.

Output

The app returns both model predictions separately, along with a class interpretation and confidence breakdown per label.

Model improvements made

  • Added sender-aware input engineering by prepending a normalized sender token to the cleaned message.
  • Strengthened the ensemble with word TF-IDF + character TF-IDF feature unions for linear models.
  • Standardized file names and inference helpers so training and deployment use the same saved model names.
  • Refactored the Flask app to return both model outputs in a single request.
  • Removed leftover Streamlit usage from project instructions and aligned setup around Flask.
