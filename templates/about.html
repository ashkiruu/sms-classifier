Project Overview

About the SMS Security Classification Project

This project addresses SMS fraud and message triage by automatically classifying incoming texts into ads, gov, notifs, otp, and spam. The system integrates two distinct architectures to provide robust security analysis.

⚠️

The Problem

Users receive mixed Tagalog-English SMS where scams and institutional alerts look similar. This classifier helps flag risks and support safer mobile interaction.

📊

The Dataset

Using specialized preprocessing, we preserve sender identity as a normalized token, allowing models to learn from both content and source patterns.

🚀

The Output

Our Flask-only application delivers dual-model predictions with detailed confidence levels and clear interpretations for end-users.

Model Improvements

  • Sender-Aware Input: Prepending normalized sender tokens to the cleaned message bodies.
  • Feature Unions: Combined word-level and character-level TF-IDF for stronger linear discrimination.
  • Architecture Sync: Standardized inference helpers so training and deployment share the same logic.
  • Unified Backend: Refactored Flask to return Ensemble and Neural Network outputs in a single atomic request.
  • Course Compliance: Removed Streamlit dependencies to focus on a pure Flask-based web implementation.

Presentation Talking Points

  • Sender Metadata: Why metadata improves discrimination between numeric spam and institutional sources.
  • Model Divergence: Comparing how the Ensemble vs Neural Network handle minority classes like gov.
  • Bilingual Nuance: Decisions made for preprocessing Taglish text and noisy punctuation patterns.
  • Uncertainty: Using confidence scores to explain why a model might be "unsure" of a classification.

Proactive Improvements

  • Multi-page Dashboard: Clean separation between analyzer, metrics, and documentation.
  • Graceful Fallbacks: Integrated error handling if TensorFlow or specific models are missing in the runtime.
  • Modular Design: Decoupled inference helpers that work across CLI, API, and Web frontends.
